## 접근
못함
## 정답
주어진 단어를 사용해 가장 짧은 펠린드롬을 만드는 문제이다.

펠린드롬인 가장 긴 접두사를 찾고, 나머지 부분을 뒤집어서 앞에 붙인다.
이때, 가장 긴 접두사가 펠린드롬인지 판단하기 위해
`롤링해시` 를 사용한다.

어떤 문자열의 앞에서부터의 해시와, 뒤에서부터의 해시가 같다면,
즉 어떤 문자열이 펠린드롬이라면,
해시를 이루는 다항식이 대칭이기 때문이다.

롤링해시의 식은 다음과 같다.

forward = forward &* base &+ value

# Q3. Sum of Scores of Built Strings
## 접근
앞에서의 해시와
뒤에서의 해시가 같으면
점수를 올린다.

## 실패
```
Input: s = "azbazbzaz"
Output: 14
Explanation: 
For s2 == "az", the longest common prefix is "az" which has a score of 2.
For s6 == "azbzaz", the longest common prefix is "azb" which has a score of 3.
For s9 == "azbazbzaz", the longest common prefix is "azbazbzaz" which has a score of 9.
For all other si, the score is 0.
The sum of the scores is 2 + 3 + 9 = 14, so we return 14.
```

즉, 부분문자열 "azbzaz" 에서
가장 긴 공통접두사는 "azb" 이므로
3점이다.
하지만 위 방법으로는
"azbzaz"와 "azbazb" 전체를 비교하기 때문에
점수를 얻을 수 없다.
즉 `앞에서 만든 문자열`과
`뒤에서 만든 문자열` 의
최장 공통 접두사를 찾아야 하는것이다.

## 정답
문자열 S의 앞자리부터 늘려가며 해시를 저장한다.

```
for i in 0..<n {
power[i + 1] = power[i] &* base
hash[i + 1] = hash[i] &* base &+ UInt64(chars[i].asciiValue!)
}
```

이제 접두사를 늘려가며,
접두사 내 모든 크기의 접두사에 대해 
원본 문자열의 접두사와 비교한다.

앞에서 모든 자리에 대한 해시를 저장했으므로
특정 구간의 해시를 계산하는것이 가능하다.

```
hash(l,r)
=
H[r+1] - H[l] * pow[r-l+1]
```

이때, 가장 긴 접두사를 찾기 위해
이분탐색을 사용하는데

mid에 대해 같은 해시라면
(현재 계산한 해시가 같다면)
(현재 계산한 접두사가 같다면)
더 긴 접두사에 대해 계산해보고

mid에 대해 다른 해시라면
(현재 계산한 해시가 다르다면)
(현재 계산한 접두사가 다르다면)
더 짧은 접두사에 대해 계산하는것이 그 원리다.

이분탐색을 쓸 수 있는 이유는
어떤 접두사가 같을 때
그것보다 짧은 접두사는 항상 같기 때문이다.

즉, 어떤 접두사가 같다면
그것보다 짧은 접두사에 대해서 같음이 보장돼 있으니
더 긴 접두사를 찾아보는 것이고

같지 않다면, 더 짧은 접두사를 살피는데
이때 이분탐색으로 구간을 줄이고, 늘려보는 것.
