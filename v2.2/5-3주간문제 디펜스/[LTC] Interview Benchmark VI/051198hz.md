# Count Subarrays With Median K
## 접근
투 포인터, 슬라이딩 윈도우를 고려했으나
두 알고리즘 모두 n개의 원소를 정렬해야 하는 구간을 갖으므로
`1 <= n <= 10^5` 에 대해 `n * log n` 이 발생.
또한, 투 포인터의 경우 부분 배열의 좌우 끝이 가변적이며
슬라이딩 윈도우의 경우 모든 원소에 대해 고려해야 하므로 `n * n * log n`
따라서 만족하는 솔루션을 낼 수 없었음

## 풀이
값 k를 기준으로, 원소가 k보다 작으면 -1, 크면 1로 치환
부분배열의 좌/우에 k값보다 작은 원소/큰 원소 의 갯수가 같거나
큰 원소의 개수가 1개 많다면, k는 중앙에 위치할 수 있음
(정렬 결과의 순서만 따지므로)

따라서 배열내 원소 k를 기준으로
우끝까지 -1,1을 누적해 해시맵에 [누적값: 등장횟수] 로 저장

좌끝까지 -1,1을 누적하며 
해시맵에 부호반대인 같은 절댓값의 누적값 
또는 1+누적값
의 빈도수 합이 정답이 된다.
