# Q3. First Missing Positive
## 접근
배열 nums = [x1, x2, x3, x4, ..., xi] 가 있고, 각 원소는 유일하다고 하자.
[1, max(nums)] 중 nums에 등장하지 않은 가장 작은 자연수를 찾을 때 다음을 생각해봤다.

##### nums가 1을 포함할 때

[4, 100 1, 400]

각 원소를 순회하며 해시맵에 기록한다.
이제 원소를 다시 순회하며, x+1, x-1의 존재 여부를 검사하고, 없는 값 중 더 작은 값을 기록하며 갱신한다.

이것이 정답인 이유는
[1, max(nums)] 의
구간 내에 존재하지 않는 가장 작은 자연수 혹은
구간 외에 존재하는 가장 작은 자연수가
원소의 양 옆에 위치하기 때문이다.

예를들어

1 2 5 9 11
일 때,

3 = 2 + 1이 존재하지 않는다.
만약 3이 존재했다고 했을 때
1, 2, 3, 5, 9, 11
4 = 3 + 1이 존재하지 않는다.
따라서 존재하지 않는 가장 작은 자연수는
이미 존재하는 원소의 양 옆에 있음을 알 수 있다.

##### nums가 1을 포함하지 않을 때

해시맵을 만들기 위해 원소를 순회할 때
1의 존재 여부를 검사한다.

1이 원소에 존재하지 않을 경우
1이 가장 작은 존재하지 않는 자연수이다.

[5,6,7,8,9] 일 때,
4 = 5 - 1 이다.
[4, 5, 6, 7, 8, 9] 일 때, 
3 = 4 - 1 이다.
[3, 100, 109, 10000] 일 때,
2 = 3 - 1 이다.

배열에 1이 존재하지 않는다는 것은
모든 nums의 원소가 1보다 크며
m = min(nums)-1 인 m이
항상 m >=1 일 것이다.
따라서 가장 작은 존재하지 않는 자연수는 1이다.

## 실패

....그러나 상기 알고리즘의 해시맵은 O(n)의 공간복잡도를 가진다....

## 정답

배열의 크기가 n일 때, 가장 작은 등장하지 않는 자연수는
[1, n]에 존재한다.

따라서 배열의 원소를 인덱스로 교환한다.

[4, 3, 1, 2]

i = 0
while
swap(i, nums[i]-1)
-> [2, 3, 1, 4]
-> [3, 2, 1, 4]
-> [1, 2, 3, 4]

이제 첫 원소부터 1과 비교해 하나씩 옮겨가며, 2와 비교...
반복하면서 불알치가 일어나는 i번째 자연수가 정답이다.
