# Number of Ways to Stay in the Same Place After Some Steps
## 접근
현재 포인터가 이전 포인터에 의해서 결정된다는 사실로부터 DP인건 파악.
그러나 DP테이블을 짜지 못함...

## 풀이
+를 오른쪽, -를 왼쪽으로 이동하는 횟수라고 하자.
n번 이동할 수 있다고 할 때, 포인터 0에 오려면
총 `0 + (n/2) - (n/2)` 이어야 0의 위치에 돌아올 수 있다.
이는 `오른쪽으로 최대 n/2번 ` 움질 수 있다는 뜻이며,
따라서 DP테이블의 크기는 n/2+1 이면 충분하다.

한 번 움직일 때, 왼/가만히/오 로 움직일 수 있다.

따라서 점화식은 다음과 같다.

```
// 가만히 있어 현재 위치에 있는 경우
현재[pos] = 이전[pos]

// 왼쪽에서 현재 위치로 이동한 경우
현재[pos] = 이전[pos - 1]

// 오른쪽에서 현재 위치로 이동한 경우
현재[pos] = 이전[pos + 1]
```
---
따라서
`현재[pos] = 이전[pos - 1] + 이전[pos] + 이전[pos + 1]`

# GCD Sort of an Array
## 접근
아래 예제에서, 양 끝을 start, end로 하는 투포인터 풀이가 가능해보였음
```
Input: nums = [7,21,3]
Output: true
Explanation: We can sort [7,21,3] by performing the following operations:
- Swap 7 and 21 because gcd(7,21) = 7. nums = [21,7,3]
- Swap 21 and 3 because gcd(21,3) = 3. nums = [3,7,21]

Input: nums = [10,5,9,3,15]
Output: true
We can sort [10,5,9,3,15] by performing the following operations:
- Swap 10 and 15 because gcd(10,15) = 5. nums = [15,5,9,3,10]
- Swap 15 and 3 because gcd(15,3) = 3. nums = [3,5,9,15,10]
- Swap 10 and 15 because gcd(10,15) = 5. nums = [3,5,9,10,15]
```

시간 초과 발생.
투포인터로는 배열이 gcd정렬 가능한지 판단할 수 없음

## 풀이

배열 내 모든 수에 대해, 공통 gcd가 있다면 연결한다.
```
[3, 15, 10]
일 때

10 = 2 * 5
15 = 3 * 5

따라서 10의 부모를 5, 15의 부모를 5로 지정.

정렬 전 배열과 정렬후 배열의 동일위치 원소에 대해
부모가 같다면, 다른 원소와 swap해 결국 그 위치로 옮길 수 있으므로
모든 원소에 대해 부모가 같은지를 비교한다.
```
