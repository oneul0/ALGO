# Q2. Find Right Interval


(start, 원래 인덱스) 배열 생성
start 기준 정렬
각 interval의 end에 대해
정렬된 배열에서 start >= end인 첫 위치를 이분 탐색
찾으면 그 원래 인덱스를 답에 저장
없으면 -1


핵심은 각 구간 [start, end]에 대해

start >= end를 만족하는 구간들 중에서 start가 가장 작은 구간의 원래 인덱스를 찾는 것입니다.

즉,

[end, ∞)

범위 안에 있는 가장 작은 start를 찾아야 합니다.

예를 들어

intervals = [[3,4],[2,3],[1,2]]
            0      1      2

각 구간에 대해 생각해보면

[3,4]

끝점이 4입니다.

start가 4 이상인 구간은 없습니다.

→ -1

[2,3]

끝점이 3입니다.

start 값들

3 (index 0)
2 (index 1)
1 (index 2)

3 이상인 것은

3

뿐입니다.

→ index 0

[1,2]

끝점이 2입니다.

2 이상인 start는

2 (index 1)
3 (index 0)

이 중 가장 작은 것은 2

→ index 1

결과

[-1,0,1]


start 값들을 정렬하자

각 interval의

(start, originalIndex)

를 저장합니다.

예)

[[3,4],[2,3],[1,2]]

↓

[(3,0),(2,1),(1,2)]

start 기준으로 정렬하면

[(1,2),(2,1),(3,0)]

이제 어떤 interval의 end가 2라면

정렬된 start들 중

2 이상인 첫 번째 원소

를 찾으면 됩니다.

즉,

lower_bound(end)

문제가 됩니다.

예)

[1,2]

end = 2

정렬된 start들

1 2 3

여기서

2 이상인 첫 번째 값

은 2

→ 원래 인덱스 1

[2,3]

end = 3

1 2 3

3 이상인 첫 번째 값

→ 3

→ 원래 인덱스 0

[3,4]
1 2 3

4 이상인 값이 없음

→ -1
